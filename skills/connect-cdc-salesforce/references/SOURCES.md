# connect-cdc-salesforce Skill Source Map

Maps each file in `skills/connect-cdc-salesforce/` to the source paths it derives from, so future syncs and human maintainers know exactly where to verify claims.

The `salesforce_cdc` input is Go source in the **public** repo `redpanda-data/connect` under `internal/impl/salesforce/`. Its user-facing connector reference is **auto-generated** in the **public** repo `redpanda-data/rp-connect-docs` (field list is a generated partial; descriptions are overridden in `docs-data/overrides.json`). The Redpanda broker-side enterprise features (the destination-topic sink claims in `redpanda-enterprise-sink.md`) are documented in the **public** repo `redpanda-data/docs` and defined in the broker config source `redpanda-data/redpanda` (`src/v/config/configuration.cc`). All repos are public — read them via the Redpanda-Github-Read MCP connector (`get_file_contents`) or `gh api`; avoid `gh search code` (rate-limited). Before writing or changing any fact, re-open the cited source and confirm exact field names, defaults, and config keys. Salesforce org setup (Connected App, Change Data Capture, Platform Events, Pub/Sub API access) is **external** to all these repos — Salesforce product behavior (the repo only ships a helper `app/setup.sh` and `README.md`). The connector was introduced in **4.90.3** (stable); verify against the current stable Connect release, not `main`.

## Deferred to live introspection / generated source (NOT drift — do not hardcode)

- **The connector field list, types, and defaults are auto-generated.** `config-reference.md` duplicates them by hand, but the citation of record is the generated partial `modules/components/partials/fields/inputs/salesforce_cdc.adoc` in rp-connect-docs (regenerate with `doc-tools generate rpcn-connector-docs`) plus the `docs-data/overrides.json` `salesforce_cdc` entry. (Spot-checked matches: `api_version=v65.0`, `stream_snapshot=true`, `snapshot_max_batch_size=2000`, `max_parallel_snapshot_objects=1`, `replay_preset=latest`, `stream_batch_size=100`, `checkpoint_cache_key=salesforce_cdc`, `checkpoint_limit=1024`, `auto_replay_nacks=true`.)
- **Salesforce org setup is external.** Connected App, OAuth Client Credentials, CDC enablement, Platform Event definitions, Pub/Sub API access, and the 24h/72h replay retention windows are Salesforce product behavior — verify against Salesforce docs. Only `app/setup.sh` and `README.md` are in-repo.
- **Broker enterprise property defaults/accepted values** (Iceberg, Tiered Storage, Cloud Topics, schema-ID validation) are broker config, not the connector. Treat the docs property partials as citation of record; upstream `src/v/config/configuration.cc`.
- **License state** — `rpk cluster license info` (broker) and Connect license status are runtime values.
