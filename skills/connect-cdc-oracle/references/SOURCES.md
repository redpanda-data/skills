# connect-cdc-oracle Skill Source Map

Maps each file in `skills/connect-cdc-oracle/` to the source paths it derives from, so future
syncs and human maintainers know exactly where to verify claims.

The skill documents the `oracledb_cdc` **input** of Redpanda Connect (LogMiner-based CDC from
Oracle Database). Two public repos are the ground truth:

- **`redpanda-data/connect`** (Go) — the connector implementation under `internal/impl/oracledb/`.
  This is an **enterprise** connector (Redpanda Community License; `license.CheckRunningEnterprise`),
  introduced in **4.83.0**. Connect is versioned by release **tag**; verify against a released tag,
  not `main`, when pinning behavior.
- **`redpanda-data/rp-connect-docs`** — the connector reference page. The field list, types, and
  defaults are **auto-generated** into a partial from the Go config; overrides live in
  `docs-data/overrides.json`. Do **not** hardcode field defaults here — defer to the generated partial.

Read both via the Redpanda-Github-Read MCP connector (`get_file_contents`) or `gh api .../contents/`.
Avoid `gh search code` (rate-limited). All connect paths verified on `redpanda-data/connect@main`;
docs paths on `redpanda-data/rp-connect-docs@main`.

## Deferred to live introspection (NOT drift — do not pin or hardcode)

- **`oracledb_cdc` field list, types, and defaults** are **auto-generated** into `modules/components/partials/fields/inputs/oracledb_cdc.adoc` (header: "This content is autogenerated. Do not edit manually. To override descriptions, use the doc-tools CLI with the `--overrides` option."). Treat that partial + `docs-data/overrides.json` as the citation of record; regenerate rather than hand-edit.
- **Connector availability version (4.83.0)** — sourced from the generated page; re-confirm per release.
- **Oracle-server setup** (ARCHIVELOG, supplemental logging, LogMiner grants, RMAN retention, Amazon RDS / OCI specifics) is **external** to both repos; the skill's SQL is validated only where the connector enforces it (`VerifyUserTables`).

## Usage

For each file being reviewed or updated, open the listed source paths first and confirm every claim
still matches. Verify Go behavior against the current released **tag** of `redpanda-data/connect` (not
`main`), and defer all `oracledb_cdc` field types/defaults to the auto-generated partial + `overrides.json`
in `rp-connect-docs` rather than trusting a static list. Oracle-server setup steps are external —
validate only what the connector enforces.
