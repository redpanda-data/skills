# Redpanda SQL (Oxla) Skill Source Map

Maps each file in `skills/sql/` to the source paths it derives from, so future syncs and human maintainers know exactly where to verify claims.

Redpanda SQL is the **Oxla** engine. The source of truth is the **private** repo `redpanda-data/oxla` — read it only via the Redpanda-Github-Read MCP connector (`search_code`, `get_file_contents`); do **not** `gh`-query or clone oxla. User-facing SQL docs are **public** and live in `redpanda-data/cloud-docs` under the `sql` module (`modules/sql/pages/`) — **not** in `redpanda-data/docs` (that repo has no `sql` module / Oxla content, verified). The public docs are fine to read via `gh api repos/redpanda-data/cloud-docs/contents/<path>`.

Oxla SQL surface = the bison grammar plus the query-planner test corpus. Before writing or changing any fact, re-open the cited source and confirm exact grammar productions, option keys, type names, and config defaults. Config defaults are version-specific — verify against the current default config, not a pinned copy.

All oxla paths below were verified to exist via the connector on the default branch (`config/Release/default_config.yml`, `config/Debug/default_config.yml`, `src/config/config_parameter_list.h`, `src/sqlparser/sql/ColumnType.h`, `src/sqlparser/sql/connection_option_names.h`, `src/sqlparser/bison_parser/bison_parser.y`, `src/catalog/kafka/conversions.cpp`, `src/catalog/iceberg_catalog_parser.cpp`, `src/catalog/storage_parser.cpp`, `src/schema/predefined_functions.cpp`/`.h`, and the `tests/UT/query_planner/cases/` tree).

## Deferred to live introspection (NOT drift — do not pin or hardcode)

- **Oxla version-specific config defaults** — values in `config/Release/default_config.yml` (buffer sizes, timeouts, port, feature-flag booleans) change across versions; re-read from the running version's config, not pinned. (`Debug` differs from `Release` only in `logging.level: DEBUG` and `feature_flags.pipeline_visualization: true`.)
- **`psql` / PostgreSQL client behavior** — `\COPY`, `PGPASSWORD`, connection-string parsing, JDBC/psycopg2/asyncpg/pgx semantics are external (PostgreSQL ecosystem), not oxla.
- **Runtime query results** — output values, row counts, `pg_typeof` results, EXPLAIN-equivalent plan text are runtime, not static.

## Usage

For each file being reviewed or updated, open the listed oxla source paths first (via the Redpanda-Github-Read connector) and confirm every claim still matches the current default branch / release. Grammar claims → `bison_parser.y`; option keys → `connection_option_names.h` + `conversions.cpp`/`iceberg_catalog_parser.cpp`/`storage_parser.cpp`; types → `ColumnType.h`; config defaults → `config/Release/default_config.yml` + `config_parameter_list.h`; SQL feature existence → the `tests/UT/query_planner/cases/` corpus. Cross-check user-facing phrasing against `redpanda-data/cloud-docs` `modules/sql/pages/`.
