# rpk-redpanda Skill Source Map

Maps each file in `skills/rpk-redpanda/` to the source paths it derives from, so future syncs and human maintainers know exactly where to verify claims.

The `rpk redpanda` family and `rpk iotune` are Go source in the **public** repo `redpanda-data/redpanda` under `src/go/rpk/pkg/cli/redpanda/` and `src/go/rpk/pkg/cli/iotune/` (both dir names verified at tag `v25.3.6`); the user-facing command reference is auto-generated in the **public** repo `redpanda-data/docs` under `modules/reference/pages/rpk/rpk-redpanda/` (one page per command) plus `modules/reference/pages/rpk/rpk-iotune.adoc`. Both are public — read them via the Redpanda-Github-Read MCP connector (`search_code`, `get_file_contents`), or `gh api .../contents/<path>`; do not guess. `rpk` is versioned: verify against the **current stable release tag** of `redpanda-data/redpanda`, not `dev`.

**Platform caveat for live verification:** most of this family is Linux-only. On a macOS/Windows rpk build, `rpk redpanda` exposes **only** the `admin` subtree (`src/go/rpk/pkg/cli/redpanda/redpanda_darwin.go` registers the darwin surface). The docs repo is the authoritative enumeration of the full (Linux) surface; a macOS `--help` sweep will miss `start`/`stop`/`mode`/`check`/`tune`/`config` and `iotune` without that being drift.

## What was verified against what (authored 2026-07-07, rpk v25.3.6, docs repo `main`)

## Deferred to live introspection (NOT drift — do not pin or hardcode)

- **The tuner list and each tuner's enabled/supported status** — varies by kernel, architecture, and cloud. Source of truth: `rpk redpanda tune list` and `rpk redpanda tune help <tuner>` on the node. The skill names example tuners only.
- `rpk redpanda <cmd> --help` flag sets and defaults — introspect on a Linux build rather than trusting a static table (this skill's flag tables are representative of v25.3.6-era docs).
- **dev-container bundled cluster properties** (`rpk redpanda start --mode dev-container`) — version-dependent preset; defer to `rpk redpanda start --mode help` / the current `rpk-redpanda-start.adoc`.
- **Node-property names and defaults** (`redpanda.*` keys set via `config set`) — broker config, not rpk; citation of record is the docs node-properties reference / `modules/reference/partials/properties/`, per release.
- `rpk iotune` benchmark results and the embedded well-known-io database contents — runtime/hardware-specific.
- Decommission-status output values and the exact failure-section wording — Admin API runtime output.
- Available logger names for `log-level set` — per-broker, via `--help-loggers`.
