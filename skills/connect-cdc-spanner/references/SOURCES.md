# connect-cdc-spanner Skill Source Map

Maps each file in `skills/connect-cdc-spanner/` to the source paths it derives from, so future syncs and human maintainers know exactly where to verify claims.

The `gcp_spanner_cdc` input is Go source in the **public** repo `redpanda-data/connect` under `internal/impl/gcp/enterprise/` (the connector and its `changestreams/` subpackage). Its user-facing field reference is **auto-generated** into the **public** repo `redpanda-data/rp-connect-docs` (the `gcp_spanner_cdc` connector page + fields partial, overridable via `docs-data/overrides.json`). Broker-side enterprise features (Iceberg, Tiered Storage, Cloud Topics, Shadowing, security) are documented in the **public** repo `redpanda-data/docs`. All repos are public — read them via the Redpanda-Github-Read MCP connector (`get_file_contents`) or `gh`; avoid `gh search code` (rate-limited). Before writing or changing any fact, re-open the cited source and confirm exact field names, defaults, and metadata keys. The connector is an Enterprise feature (`license.CheckRunningEnterprise`) introduced in **4.56.0**; verify against the current Connect release, not `main`.

Scope note: the **field list and defaults are auto-generated** — the citation of record is the generated fields partial in `rp-connect-docs` plus any override in `docs-data/overrides.json`, both derived from the Go `ConfigSpec`. Do not hardcode/pin field defaults. GCP Spanner setup (change-stream DDL, IAM, dialects, retention) is **external Google Cloud behavior**, not Redpanda source.
